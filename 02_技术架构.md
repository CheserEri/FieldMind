# 技术架构

版本：v1.0　　日期：2026-05-01

## 1. 系统总体架构

系统分为三层：田间硬件感知层、主机 AI 处理层、执行反馈层。三层通过 WiFi（TCP/HTTP）通信，主机端全部本地运行，无云端依赖。

| 层级 | 组成 | 通信协议 |
|------|------|---------|
| 田间硬件感知层 | ESP32-S3 + 摄像头 + 麦克风 + 多传感器 + 继电器 | HTTP POST / WebSocket |
| 主机 AI 处理层 | Ollama + 视觉模型 + 自研音频模型 + Agent LLM | localhost REST API |
| 执行反馈层 | 继电器组 + 手机推送 + SQLite 日志 | GPIO / HTTP Webhook |

## 2. 硬件层技术规格

### 2.1 主控芯片

- **型号**：ESP32-S3（推荐 ESP32-S3-WROOM-1）
- **CPU**：Xtensa LX7 双核 240MHz，带向量加速器（可跑轻量 TFLite 模型）
- **RAM**：512KB SRAM + 外接 8MB PSRAM（摄像头帧缓冲必需）
- **通信**：WiFi 802.11 b/g/n，蓝牙 5.0
- **工作电压**：3.3V，支持太阳能充电管理模块（CN3791 或 TP4056）

### 2.2 传感器模组

| 传感器 | 型号 | 接口 | 精度/范围 | 采集策略 |
|--------|------|------|----------|---------|
| 摄像头 | OV2640 200万像素 | DVP / SPI | JPEG 压缩输出 | 定时 30 分钟 / 事件触发 |
| 麦克风 | INMP441 MEMS | I2S | 24-bit 44.1kHz | 持续采集，滑动窗口分析 |
| 土壤湿度 | Capacitive v2.0 | ADC | 0–100% 相对湿度 | 每 5 分钟读取，低于阈值上报 |
| 温湿度 | DHT22 | OneWire | ±0.5°C / ±2% RH | 每 10 分钟读取 |
| 光照 | BH1750 | I2C | 1–65535 lux | 每 10 分钟读取 |

### 2.3 执行器模组

| 执行器 | 驱动方式 | GPIO | 安全等级 | 单次最大时长 |
|--------|---------|------|---------|------------|
| 灌溉水阀 | 5V 继电器 → 电磁阀 | GPIO 4 | 低（自动执行） | 30 分钟 |
| 遮阳网 | 继电器 → 直流电机驱动 | GPIO 5/6 | 低（自动执行） | 5 分钟 |
| 施肥泵 | 继电器 → 蠕动泵 | GPIO 7 | 高（需确认） | 10 分钟 |
| 喷药阀 | 继电器 → 电磁阀 | GPIO 8 | 高（需确认） | 15 分钟 |

## 3. 主机 AI 处理层

### 3.1 视觉模型

- **框架**：Ollama（本地推理）
- **推荐模型**：LLaVA-1.6-7B（显存 ≥ 8GB）或 Moondream2（显存 ≥ 4GB）
- **Prompt 模板**：

  > 你是农业专家。请描述这张图片中作物的生长状态、叶片颜色、是否存在病斑或虫害迹象，输出不超过 100 字的结构化描述。

- **输出格式**：纯文字，直接拼入 Agent 上下文

### 3.2 自研环境音分类模型

- **基础架构**：MobileNetV2 音频特征提取层（迁移学习）
- **输入**：梅尔频谱图（128×128，1 秒窗口，0.5 秒步长）
- **输出类别（初版 8 类）**：微风 / 强风 / 小雨 / 暴雨 / 正常虫鸣 / 异常虫鸣密度 / 设备异常音 / 环境静默
- **训练框架**：TensorFlow / Keras，量化导出 TFLite INT8
- **部署位置**：主机端 Python 推理服务（FastAPI），预留 ESP32-S3 边缘推理接口

### 3.3 Agent 决策核心

- **LLM**：Qwen2-7B-Instruct 或 Llama3-8B（Ollama 部署）
- **上下文组装顺序**：
  1. `[系统提示词]` 农业专家角色 + 执行权限说明
  2. `[视觉描述]` 来自视觉模型的作物状态文字
  3. `[环境音分析]` 来自自研模型的音频类别 + 置信度
  4. `[传感数据]` 土壤湿度 X%，温度 X°C，光照 X lux，时间戳
  5. `[历史上下文]` 近 3 次操作记录（可选）

- **输出格式（结构化 JSON）**：

  ```json
  {"action": "irrigate", "reason": "土壤湿度28%低于阈值", "risk": "low", "duration": 600}
  ```

## 4. 数据流说明

| 数据类型 | 采集端 | 处理端 | 输出 | 频率 |
|---------|--------|--------|------|------|
| 图像帧 | ESP32 摄像头 | 视觉模型 | 作物状态文字 | 30 分钟 / 事件 |
| 环境音频 | ESP32 麦克风 | 自研音频模型 | 环境类别 + 置信度 | 实时滑窗 |
| 传感器数值 | ESP32 ADC/I2C | 主机解析模块 | 结构化键值对 | 5–10 分钟 |
| 执行指令 | Agent 输出 | ESP32 继电器 | GPIO 高低电平 | 按需触发 |
| 操作日志 | 主机记录 | SQLite | 训练数据回流 | 每次执行 |
