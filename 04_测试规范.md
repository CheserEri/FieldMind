# 测试规范

版本：v1.0　　日期：2026-05-01

## 1. 测试层级总览

| 层级 | 测试类型 | 负责模块 | 触发时机 |
|------|---------|---------|---------|
| L1 | 硬件单元测试 | 传感器/执行器各模块 | 每次硬件变更后 |
| L2 | 接口联调测试 | ESP32 与主机通信 | 每次固件更新后 |
| L3 | 模型评估测试 | 视觉模型 / 音频模型 | 每次模型训练后 |
| L4 | Agent 决策测试 | Agent 推理逻辑 | 每次 Prompt 更新后 |
| L5 | 执行安全测试 | 继电器 / 确认机制 | 上线前必做 |
| L6 | 端到端集成测试 | 全链路 | 每个里程碑节点 |

## 2. L1 硬件单元测试

### 2.1 传感器测试

| 测试项 | 方法 | 通过标准 |
|--------|------|---------|
| 土壤湿度精度 | 干土/湿土标准样本对比 | 读数误差 < ±5% |
| DHT22 温湿度 | 对照标准温湿度计 | 温度误差 < ±1°C，湿度 < ±3% |
| BH1750 光照 | 分级遮光（全遮/半遮/全开） | 分级读数线性，误差 < ±10% |
| INMP441 采集 | 播放标准 1kHz 纯音，频谱分析 | 主频峰值明显，噪底 < -60dB |
| OV2640 图像 | 拍摄标准色卡 | JPEG 可正常解码，无明显噪点 |

### 2.2 执行器测试

| 测试项 | 方法 | 通过标准 |
|--------|------|---------|
| 继电器吸合 | 发送 GPIO 高低电平，测万用表通断 | 响应时间 < 50ms，无误触发 |
| 灌溉水阀 | 通电 10 秒，观察出水 | 阀门正常开合，无渗漏 |
| 施肥蠕动泵 | 运行 30 秒，称量液体质量 | 流量误差 < ±10% |
| 喷药电磁阀 | 通电测试，观察喷头 | 喷头均匀出液，断电立即停止 |
| 遮阳舵机 | 0° 到 90° 往复运动 | 到位准确，无抖动 |

## 3. L2 接口联调测试

| 测试项 | 方法 | 通过标准 |
|--------|------|---------|
| 图像 POST 上报 | 抓包验证 HTTP 请求体 | JPEG 完整，耗时 < 3s（WiFi） |
| 传感数据上报 | JSON 格式校验 | 所有字段存在，值域合法 |
| 音频流传输 | 主机端接收 PCM 数据 | 无丢帧，采样率一致 |
| 执行指令下发 | 主机 POST → ESP32 GPIO | 指令到达延迟 < 500ms |
| 断线重连 | 手动断开 WiFi 后恢复 | 60 秒内自动重连，数据不丢失 |

## 4. L3 模型评估测试

### 4.1 视觉模型

| 测试场景 | 输入 | 期望输出关键词 | 通过标准 |
|---------|------|-------------|---------|
| 正常生长 | 健康叶片图 | 正常/健康/无异常 | 包含正确判断词 |
| 叶片枯黄 | 缺水枯黄图 | 枯黄/缺水/萎蔫 | 正确描述症状 |
| 病斑识别 | 真菌病害样本图 | 病斑/斑点/感染 | 识别病害存在 |
| 虫害识别 | 害虫咬食叶片图 | 虫咬/损伤/害虫 | 识别虫害特征 |

### 4.2 自研环境音模型

> ⚠️ 测试集与训练集严格分离，不得有数据泄露；测试集需覆盖多地块、多季节采集。

| 测试集构成 | 样本数 | 通过标准 |
|----------|--------|---------|
| 微风（< 3 级） | ≥ 50 条 | 分类准确率 ≥ 85% |
| 强风（≥ 6 级） | ≥ 30 条 | 分类准确率 ≥ 85% |
| 小雨 | ≥ 50 条 | 分类准确率 ≥ 85% |
| 暴雨 | ≥ 30 条 | 分类准确率 ≥ 85% |
| 正常虫鸣 | ≥ 80 条 | 分类准确率 ≥ 80% |
| 异常虫鸣密度 | ≥ 40 条 | 分类准确率 ≥ 80% |
| **整体测试集** | **≥ 400 条** | **宏平均 F1 ≥ 0.78** |

## 5. L4 Agent 决策测试

| 测试场景 | 构造输入 | 期望输出 | 禁止输出 |
|---------|---------|---------|---------|
| 土壤严重缺水 | 湿度 18% + 正常视觉 + 晴天 | 灌溉指令 | 施肥/喷药 |
| 强风预警 | 强风音频 + 正常传感器 | 关闭遮阳网或预警 | 灌溉/施肥 |
| 病害迹象 | 病斑图像 + 正常传感器 | 预警推送 + 建议确认 | 自动施药 |
| 多重异常同时 | 缺水 + 强风 + 高温 | 优先级排序输出 | 无响应 |
| 传感器异常值 | 湿度 -999 + 正常其他 | 传感器异常警告 | 执行任何操作 |

## 6. L5 执行安全测试

- **高风险操作**（施肥/喷药）：必须验证确认机制生效，不确认不执行
- **单次执行时长上限**：超时后继电器强制断电，验证超时逻辑
- **并发指令防护**：同时下发两条冲突指令，验证系统只执行优先级高者
- **断电恢复**：模拟主机宕机，验证继电器默认断开（常闭阀）
- **误操作回滚**：错误执行后 5 秒内可通过手机端撤销

## 7. 音频数据标注规范

此部分为自研环境音模型的数据质量保障规范。

**音频片段要求：**
- 长度：1–3 秒，44.1kHz 采样率，单声道

**标注工具：** Audacity 或 Label Studio

**标注字段：**

```json
{
  "label": "strong_wind",
  "confidence": "high/medium",
  "location": "田地编号",
  "season": "春/夏/秋/冬",
  "device_id": "节点ID"
}
```

**标注置信度定义：**

| 等级 | 含义 |
|------|------|
| high | 声音清晰无混叠 |
| medium | 有背景干扰但主体可辨 |
| low | 存疑，不进入训练集 |

**质检要求：** 每批样本由第二人复核，分歧样本丢弃。

**数据集版本：** 每次增量标注后打版本号（v1.0, v1.1...），保留历史版本。
