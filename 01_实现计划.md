# 实现计划

版本：v1.0　　日期：2026-05-01　　状态：草稿

## 1. 项目目标

构建一套低成本、全本地部署的农业田间智能系统，实现多传感器环境感知、自研深度学习环境音分类、AI Agent 决策推理，以及继电器驱动的自动化执行（灌溉、遮阳、施肥、喷药），覆盖从感知到执行的完整闭环。

## 2. 总体阶段规划

| 阶段 | 名称 | 周期 | 核心交付物 |
|------|------|------|-----------|
| P0 | 硬件验证 | 第 1–2 周 | ESP32 与传感器联通，数据上报主机 |
| P1 | 感知层搭建 | 第 3–5 周 | 视觉模型接入，传感数据解析，基础 Agent 对话 |
| P2 | 环境音模型 | 第 6–12 周 | 数据采集标注，模型训练与部署 |
| P3 | 执行层接入 | 第 13–15 周 | 继电器控制，安全确认机制，手机推送 |
| P4 | 闭环优化 | 第 16–20 周 | 数据回流，模型迭代，长期日志分析 |

## 3. 各阶段详细任务

### P0 硬件验证（第 1–2 周）

- ESP32-S3 开发板上电，配置 WiFi 连接主机
- 接入土壤湿度传感器（模拟量读取），验证 ADC 精度
- 接入温湿度（DHT22）与光照传感器（BH1750），I2C 通信验证
- 摄像头模组（OV2640）初始化，JPEG 压缩编码测试
- 麦克风（INMP441）I2S 接口联调，原始 PCM 数据上报主机
- 继电器模块接线，GPIO 高低电平控制验证

### P1 感知层搭建（第 3–5 周）

- 主机端搭建 Ollama，部署 LLaVA / Moondream 视觉模型
- 图像帧接收与推理流水线，输出结构化作物状态描述
- 传感数据解析模块，格式化为 Agent Prompt 片段
- 基础 Agent 接入（Qwen2-7B / Llama3-8B），多路文字融合测试
- 主机端 HTTP Server，ESP32 POST 上报接口开发

### P2 环境音模型（第 6–12 周）

- **第 6–7 周**：田间麦克风部署，采集不同天气/季节背景音频
- **第 8 周**：音频标注规范制定，建立标注工作流（见[测试规范](04_测试规范.md#7-音频数据标注规范)）
- **第 9–10 周**：基于迁移学习（MobileNet 音频特征层）训练分类模型
- **第 11 周**：模型量化（TFLite INT8），部署至主机端推理
- **第 12 周**：准确率验证，边界案例测试，迭代第一版

### P3 执行层接入（第 13–15 周）

- 继电器控制协议设计，Agent 输出映射执行指令
- 安全确认机制：高风险操作推送微信/短信，等待确认后执行
- 执行操作日志写入本地数据库（SQLite）
- 手机端简易看板（可选：Flask + 微信推送）

### P4 闭环优化（第 16–20 周）

- 操作结果反馈采集，标注执行前后作物状态变化
- 新增音频样本回流训练管线，自动重训分类模型
- 季节性数据趋势分析，Agent 提示词持续优化
- 多节点扩展测试（≥ 3 个田间节点并行运行）

## 4. 里程碑与验收标准

| 里程碑 | 验收标准 | 目标日期 |
|--------|---------|---------|
| M1 硬件就绪 | 所有传感器数据稳定上报，误差率 < 5% | 第 2 周末 |
| M2 感知闭环 | Agent 能根据图像 + 传感数据输出合理文字判断 | 第 5 周末 |
| M3 音模型 v1 | 环境音分类准确率 ≥ 80%（测试集） | 第 12 周末 |
| M4 执行闭环 | 灌溉/遮阳自动执行，喷药确认后执行，无误操作 | 第 15 周末 |
| M5 自迭代 | 新数据入库后模型自动重训，准确率持续提升 | 第 20 周末 |

## 5. 风险与应对

| 风险项 | 影响 | 应对措施 |
|--------|------|---------|
| WiFi 信号覆盖不足 | 数据丢包，执行指令延迟 | 增加 WiFi 中继或改用 LoRa 模块 |
| 环境音训练数据不足 | 分类模型准确率低 | 与农业院校合作，扩大采集地块 |
| 执行器误操作 | 烧苗或农药滥用 | 高风险指令强制人工确认，设置单次用量上限 |
| Ollama 主机算力不足 | 推理延迟高，响应慢 | 降低视觉模型参数量，或升级 GPU 显存 |
| 传感器漂移老化 | 数据失真导致错误决策 | 定期校准，设置合理值域过滤（见[错误码规范](05_错误码规范.md)） |
